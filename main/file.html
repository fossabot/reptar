File#

The heart of reptar is the File class which allows you to interact with your data in a storage-agnostic way.

Storage types#

Every file type comes with its own list of advantages and disadvantages leading to best use cases. We let the user decide what file format to use and have reptar provide a consistent way of interacting with the data. Reptar currently supports the following file types:

  • exdir,

  • JSON,

  • npz.

Any data stored in this format can be interacted with through a File object.

class reptar.File(file_path, mode='r', allow_remove=False, plugins=None, from_dict=None)[source]

Create, store, and access data from a variety of formats.

Parameters:

  • file_path (str) – Path to a file supported by reptar. If it does not exist, then one will be created if possible.

  • mode (str, default: 'r') – A file mode string that defines the read/write behavior.

  • allow_remove (bool, default: False) – Allow the removal of exdir groups in 'w' operation.

  • plugins (list, default: None) – A list of instantiated exdir plugins.

  • from_dict (dict, default: None) – Load data from a dictionary.

Concepts#

All data is stored in key-value pairs. The key provides a human-readable label to communicate the meaning of the value. For example, energy_pot and geometry refers to potential energy and Cartesian coordinates, respectively. More information about key definitions can be found here.

The key also tells reptar where the data is stored. Many file types (e.g., exdir and JSON) support nested objects which allow us to organize our data into groups. A group is essentially a collection of related data (e.g., a directory) that could also contain other groups. We specify keys for nested groups the same way we do directory paths where the file itself is considered the root: group1, /group1, and group1/nested_group are all valid group keys.

For example, suppose we have a file that contains two molecular dynamics (MD) runs of a box of water. We can visualize a group tree where we have two groups (run_1 and run_2) that contain the geometry and energy_pot from both MD simulations. If we wanted to get the potential energies of both runs we would use /run_1/energy_pot and /run_2/energy_pot. Note that the / prefix is not required (run/energy_pot works as well).

.
└── water-box-md
    ├── run_1
    │   ├── energy_pot
    │   ├── geometry
    ├── run_2
    │   ├── energy_pot
    │   └── geometry

Retrieving data#

Reptar provides a single method, get(), to retrieve data from any of the supported file types. This can return any nested data stored in a loaded file including other groups.

File.get(key, as_memmap=False)[source]

Retrieve data.

Parameters:

  • key (str) – Key of the desired data. Nested keys should be separated by /.

  • as_memmap (bool, default: False) – Keep NumPy memmap (from exdir files) instead of converting to arrays.

Examples

>>> rfile.get('/prod_1/charge')
0
>>> rfile.get('energy_scf')
-12419.360138637763

You can also get a list of all non-group keys using get_keys().

File.get_keys(group_key)[source]

A list of keys in a group.

Does not include keys of nested groups.

Parameters:

group_key (str) – Key specifying which group to get the data keys from.

Returns:

A list of all available keys not leading to another group.

Return type:

list

Adding data#

Reptar provides a single method, put(), to store data in any of the supported file types.

File.put(key, data)[source]

Put data to file in a specific location.

Note that there is some data postprocessing using simplify_iter_data().

Parameters:

  • key (str) – Key of the desired data (including parent). Nested keys should be separated by /.

  • data (obj) – Data to add to file.

Note that exdir groups must be initialized first with create_group() before adding data.

File.create_group(key)[source]

Initialize/create an exdir group with the specified key.

Parameters:

key (str) – Key of the desired data (including parent). Nested keys should be separated by /.

Saving#

Besides exdir, all other file types need to be explicitly saved using the following method.

File.save(json_prettify=True)[source]

Saves non-exdir files.

Parameters:

json_prettify (bool) – Indents JSON objects if True. If false the JSON file is only one line.

File conversion#

A group from any file type can be directly converted to a dictionary using as_dict().

File.as_dict(group_key)[source]

Get a group as a dictionary.

Parameters:

group_key (str) – Desired group.

Returns:

The desired group as a dictionary.

Return type:

dict

Dictionaries and can also be used to populate a File object with any specified file type.
