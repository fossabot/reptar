Creator#

Reproducibility of computational chemistry research is the driving force of reptar. Open data is a crucial for making this happen, but established solutions often cannot support evolving research interests. In order to make this as easy as possible, reptar provides a creator class that handles turning calculations into reproducible data. By providing a customizable parser framework, reptar will automatically take care of storing the essential information while leaving the option for the user to easily parse their own data.

Group creation#

First, a data object needs to be loaded into the creator. Reptar provides two ways to do this: by initializing a creator object with a data object, or by using the load() method.

class reptar.creator(rfile=None)[source]

Create groups from computational chemistry data.

Parameters:

rfile (reptar.File, default: None) – An initialized reptar File.

creator.load(file_path, mode='r', allow_remove=False, plugins=None)[source]

Load a reptar file for creating/adding information.

Parameters:

  • file_path (str) – Path to a file supported by reptar. If it does not exist, then one will be created if possible.

  • mode (str, default: 'r') – A file mode string that defines the read/write behavior.

  • allow_remove (bool, default: False) – Allow the removal of exdir groups in w operation.

  • plugins (list, default: None) – A list of instantiated exdir plugins.

Reptar can then parse information and create a group by specifying the paths to the relevant output files.

Types of output files#

There are currently three supported types of output files used (that also depend on the package).

out_path#

The default log file generated by the software package.

geom_path#

A file containing a single (usually the initial) geometry.

traj_path#

The trajectory specifying the structural evolution during a job (e.g., geometry optimization or MD simulation).

Note

If both geom_path and traj_path are provided, the structure defined in geom_path is considered the first structure.

Additional data#

Other data cannot be easily parsed from output files. We provide methods to easily add this information.

creator.ids(group_key, entity_ids, comp_ids)[source]

Add entity_ids and comp_ids to a group.

Parameters:

  • group_key (str) – Key to the group to add IDs to (including parent).

  • entity_ids (numpy.ndarray, ndim: 1) – A uniquely identifying integer specifying what atoms belong to which entities. Entities can be a related set of atoms, molecules, or functional group.

  • comp_ids (numpy.ndarray, ndim: 1) – Relates entity_id to a fragment label for chemical components or species. Labels could be WAT or h2o for water, MeOH for methanol, bz for benzene, etc. There are no standardized labels for species. The index of the label is the respective entity_id.

Returns:

The reptar file group.

Return type:

obj

creator.definitions(definitions=None)[source]

Add definitions of data to the file.

The base definitions are included by default. There should be no overlap of any definitions. If there is, precedence is given in order of definitions with base always being first.

Parameters:

definitions (list [str], default: None) – Paths to data definition YAML files to congregate. Only the name (not a path) is required for ones provided by reptar.

Notes

Reserved definition YAML files: base, md, molprop, pes, qc, sampl, solv, xtb.
