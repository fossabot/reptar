reptar.calculators.drivers module#

class reptar.calculators.drivers.driverENGRAD(Z, R, E, G, worker, worker_args, n_cpus, n_cpus_worker=4, start_slice=None, end_slice=None)[source]#

Bases: object

Supervisor of energy+gradient workers.

Creates and manages ray tasks using specified worker.

Parameters:

  • Z (numpy.ndarray, ndim: 1) – Atomic numbers of the atoms with repsect to R. Should have shape (i,) where i is the number of atoms in the system.

  • R (numpy.ndarray, ndim: 3) – Cartesian coordinates of all structures in group. Should have shape (j, i, 3) where j is the number of structures. Units are in Angstroms.

  • E (numpy.ndarray, ndim: 1) – Total electronic energies of all structures in R. Energies that need to be calculated should have NaN in the element corresponding to the same index in R. Should have shape (j,). Units are in Hartrees.

  • G (numpy.ndarray, ndim: 3) – Atomic gradients of all structures in R. Gradients that need to be calculated should have NaN in the corresponding elements. Should have the same shape as R. Units are in Hartrees/Angstroms

  • worker (callable) – The desired worker function to compute energy and gradients. It should be the same as any previous calculations. The ray.remote decorator will be applied automatically.

  • worker_args (tuple, ndim: 1) – The other required arguments for the worker function after Z, R, and R_idxs.

  • n_cpus (int) – Total number of CPUs to use for all workers (same as num_cpus when initalizing ray).

  • n_cpus_worker (int, default: 4) – Number of CPUs to dedicate to each worker.

  • start_slice (int, default: None) – Trims R to start at this index.

  • end_slice (int, default: None) – Trims R to end at this index.

run()[source]#

Run the workers.

Returns:

  • numpy.ndarray – The total electronic energy array, E, after all computations.

  • numpy.ndarray – The atomic gradients array, E, after all computations.

reptar.calculators.psi4_workers module#

reptar.calculators.psi4_workers.psi4_engrad(Z, R, R_idxs, charge, mult, method, options, threads, mem)[source]#

Worker function for computing total electronic energy and atomic gradients using Psi4.

Parameters:

  • Z (ray.ObjectRef of numpy.ndarray, ndim: 1) – Atomic numbers of the atoms with repsect to R.

  • R (ray.ObjectRef of numpy.ndarray, ndim: 3) – Cartesian coordinates of all structures in group. This includes unused structures.

  • R_idxs (numpy.ndarray, ndim: 1) – Indices of the structures from R to compute energies and gradients for.

  • charge (int) – Total molecular charge.

  • mult (int) – Total molecular multiplicity.

  • method (str) – Specifies the Psi4 method used for the gradient. For more information, please see the Psi4 documentation for your specific version.

  • options (dict) – Psi4 control keywords using the PsiAPI format. Some common ones are basis, e_convergence, d_convergence, and reference.

  • threads (int) – Number of threads for Psi4. This is almost always the number of cores being used for the worker. For more information, see the documentation.

  • mem (int, float, str) –

    The amount of memory available. For more information, see the documentation.

Returns:

  • numpy.ndarray – R_idxs

  • numpy.ndarray – Total electronic energy of computed structures in the same order as R_idxs. Units of Hartree.

  • numpy.ndarray – Atomic gradients of computed structures in the same order as R_idxs. Units of Hartree/Angstrom.
